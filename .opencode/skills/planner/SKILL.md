# 策划技能

## 触发场景

当需要将模糊的需求转化为具体的功能规格时使用。

## 执行步骤

### 1. 需求理解
- 分析用户的愿景描述
- 提取核心功能点
- 识别技术约束

### 2. 现状分析
- 查看项目结构
- 了解现有功能
- 评估技术可行性

### 3. 文档输出
- 创建 `docs/requirements/` 目录
- 输出功能规格文档

## 文档模板

```markdown
# [功能名称] 需求文档

## 1. 背景与目标
### 1.1 业务背景
### 1.2 目标用户
### 1.3 预期效果

## 2. 功能详情
### 2.1 功能描述
### 2.2 用户故事
### 2.3 交互流程

## 3. 技术方案
### 3.1 架构设计
### 3.2 数据模型
### 3.3 接口设计
### 3.4 关键算法

## 4. 实现要求
- [ ] 后端实现
- [ ] 接口开发
- [ ] 数据库变更

## 5. 测试用例
### 5.1 正常流程
### 5.2 边界条件
### 5.3 异常处理

## 6. 验收标准
- [ ] 功能验收
- [ ] 性能验收
- [ ] 安全验收

## 7. 风险评估
### 7.1 技术风险
### 7.2 进度风险

## 8. 附录
```

## 输出位置

`docs/requirements/<功能名>.md`
