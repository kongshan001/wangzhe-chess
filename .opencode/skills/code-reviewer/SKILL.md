# 代码审核技能

## 触发场景

当代码实现完成后，进行质量审核时使用。

## 审核维度

### 1. 代码质量
- 可读性
- 可维护性
- 代码复杂度
- 命名规范
- 注释完整性

### 2. 安全性
- 输入验证
- SQL 注入
- XSS 攻击
- 权限控制
- 敏感数据处理

### 3. 性能
- 算法效率
- 数据库查询
- 内存使用
- 并发处理

### 4. 测试
- 单元测试覆盖
- 边界条件测试
- 异常场景测试

## 审核流程

### 1. 静态检查
```bash
ruff check src/
mypy src/
```

### 2. 代码审查
- 查看变更文件
- 逐文件审核
- 记录问题

### 3. 报告输出
- 输出到 `docs/reviews/`
- 标注问题严重程度
- 提供修复建议

## 问题等级

| 等级 | 说明 | 处理方式 |
|------|------|----------|
| 严重 | 必须修复 | 阻塞流程 |
| 警告 | 建议修复 | 可继续 |
| 建议 | 优化建议 | 可选 |

## 输出模板

```markdown
# 代码审核报告

**审核时间**: YYYY-MM-DD HH:MM
**审核范围**: [文件列表]

## 审核结果

### 🔴 严重问题 (必须修复)
1. [问题描述]
   - 文件: `path/to/file.py:行号`
   - 建议: [修复建议]

### 🟡 警告 (建议修复)
1. [问题描述]

### 🟢 优化建议
1. [优化建议]

## 总体评价
[通过/需要修改]

## 下一步行动
- [ ] 修复严重问题
- [ ] 处理警告项
```
