# 王者之奕 - 多角色协作系统

> 版本: 1.0 | 更新: 2026-02-21

## 🎭 角色定义

### 1. 策划 (Game Designer)
**职责**: 持续提出游戏需求和功能改进

**工作内容**:
- 分析游戏平衡性
- 提出新英雄/羁绊设计
- 定义玩法规则
- 编写需求文档

**输出**:
- `docs/requirements/*.md` - 需求文档
- `docs/balance-analysis.md` - 平衡分析
- `config/heroes.json` - 英雄配置
- `config/synergies.json` - 羁绊配置

---

### 2. 程序 (Developer)
**职责**: 使用 opencode/claude code 高效完成需求

**工作流程**:
1. 接收策划需求
2. 使用 opencode 实现功能
3. 编写代码和注释
4. 提交代码审核

**输出**:
- `src/**/*.py` - 源代码
- `docs/api/*.md` - 接口文档
- `docs/architecture.md` - 架构文档

---

### 3. 代码审核人 (Code Reviewer)
**职责**: 审核代码质量和安全性

**审核内容**:
- 代码规范
- 性能问题
- 安全漏洞
- 架构合理性
- 类型安全

**输出**:
- `docs/code-review/*.md` - 审核报告
- 代码优化建议

---

### 4. QA (Quality Assurance)
**职责**: 编写测试，确保100%覆盖率

**测试类型**:
- 单元测试
- 集成测试
- 端到端测试
- 性能测试

**输出**:
- `tests/**/*.py` - 测试代码
- `docs/test-coverage.md` - 覆盖率报告

---

## 🔄 工作流程

```

│  策划   │ ─────────> │  程序   │ ─────────> │代码审核 │
└─────────┘            └─────────┘            └────┬────┘
     ↑                                            │
     │                    反馈                    │
     └────────────────────────────────────────────┘
                              │

```
