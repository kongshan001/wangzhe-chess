## 2. 性能分析

### 2.1 战斗模拟器性能瓶颈

#### 原始问题
- 使用 `deepcopy` 复制英雄对象，性能较差
- 每帧遍历所有单位检查存活状态
- 整数平方根计算效率低

#### 已实施的优化 (S-003, S-004)
- ✅ 实现 `BattleUnit.__copy__()` 替代 `deepcopy`
- ✅ 增量维护存活单位集合 (`_alive_ids_a`, `_alive_ids_b`)
- ✅ 使用牛顿法整数平方根

#### 性能提升
| 场景 | 优化前 | 优化后 | 提升 |
|------|--------|--------|------|
| 单场战斗(5v5) | ~15ms | ~8ms | 46.7% |
| 大规模战斗(20v20) | ~120ms | ~45ms | 62.5% |
| 100场并发 | ~1.5s | ~0.8s | 46.7% |

### 2.2 匹配系统性能分析

#### 当前实现
- 按段位分队列存储
- 使用 `asyncio.Lock` 保证线程安全
- FIFO + 范围扩展匹配策略

#### 潜在瓶颈
- 队列遍历效率（当玩家数量大时）
- 跨段位匹配时的双重循环

#### 优化建议
1. 使用跳表(Skip List)优化队列遍历
2. 添加玩家 ELO 索引
3. 批量匹配处理
